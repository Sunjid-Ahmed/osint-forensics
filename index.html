IRON LAW — NEVER VIOLATE

🔍 OSINT CTF — Step by Step Methodology
INTERMEDIATE
Analyze every pixel of provided media — zero assumptions
Images: read all text, signs, vehicle plates, architecture. Extract EXIF immediately: exiftool image.jpg. Check GPS, software, author. Look for hidden data. Never skip this step.
Reverse image search all visual elements aggressively
Google Images → TinEye → Yandex (best for faces) → Bing. Crop individual elements — a distinctive sign, building facade, or logo often leads directly to exact location. Try both full image and crops.
Hunt usernames across all platforms systematically
Run: sherlock username and maigret username --html. Cross-reference: same username on LinkedIn + Reddit + GitHub = full profile reconstruction. Read all post history and bio carefully.
Check Wayback Machine and archives for deleted content
echo target.com | waybackurls — old profiles and deleted posts survive for years. Google cache may have very recent content not yet archived. Check all snapshots, not just the most recent.
DNS and certificate intelligence for domain targets
subfinder -d domain.com -all + crt.sh lookup. Staging subdomains, old dev environments, and admin panels often contain the flag or next pivot point that production hides.
Breach data and social correlation for person targets
Check holehe email@domain.com for platform presence. Search IntelX/Dehashed for leaked data. Correlate accounts across platforms to build full identity profile.
Pivot aggressively — document every finding with timestamp
Every data point leads to three more. Build a visual map in Maltego or even on paper. Revisit dead ends after each new discovery — context changes what was previously meaningless data.

🔬 Forensics CTF — Complete Workflow
INTERMEDIATE
Identify file type — never trust the extension
file unknown_file → xxd unknown_file | head -4 for magic bytes. Extensions lie; headers don't. Common CTF trick: image with wrong extension, corrupted header, or data appended after EOF.
Extract embedded data from all containers
binwalk -e file finds embedded archives and files. foremost file -o out/ carves by file signatures. strings file | grep -i "flag{" for instant wins. Try 7z x file on anything.
Steganography analysis — methodical approach
steghide info image.jpg then extract. For PNG: zsteg image.png checks all LSB planes. stegsolve for visual bit-plane analysis. Try common passwords: "", "password", "1234", challenge name.
PCAP analysis — layer by layer
File → Export Objects → HTTP to reconstruct files. Follow each TCP stream (Ctrl+Alt+Shift+T). Filter http.request.method == "POST". Decode base64 from traffic. Check all protocols for cleartext credentials.
Memory dump analysis — structured investigation
vol3 -f dump.dmp windows.info first. Then pslist → cmdline → netscan → clipboard. Flags are often in process memory, clipboard data, or environment variables. Dump suspicious processes.
Disk image analysis — use Autopsy systematically
Open in Autopsy → run all ingest modules. Check: Deleted Files, Documents, Browser History, Registry. strings disk.img | grep -i "flag{" for quick wins. Look for files with unusual timestamps.

📖 How to Read a Writeup Effectively
Attempt the challenge first — even partially
Try for at least 30 minutes before reading the writeup. Compare your approach. The gap between your thinking and the solution is where learning happens.
Understand the WHY, not just the HOW
Ask: "Why did they run this command here?" "How did they know to look in that location?" The reasoning pattern matters more than the exact commands used.
Reproduce every step in your own environment
Download the challenge file and follow along hands-on. Watching someone else solve it builds almost no skill. Doing it yourself builds everything.
Extract reusable techniques to your notes
Maintain notes in Obsidian or CherryTree. Save commands, techniques, and tool flags you didn't know. Build your own personal cheatsheet over months.
Write and publish your own writeup
Writing forces clarity of understanding. Even failed approaches are valuable. Publish on Medium or GitHub — it builds your portfolio visible to employers and the community.

🎭 Investigation OPSEC Architecture
ADVANCED
// ISOLATION LAYERS
Dedicated OS — never your daily machine
Tails OS (amnesic, no traces), Whonix (routes all traffic via Tor), or clean Kali VM. Snapshot before investigation, revert after.
VPN → Tor Browser chain
VPN first (Mullvad or ProtonVPN), then Tor. Tor entry node sees VPN IP, not your real IP. Use only for investigation activities.
Sock puppet accounts for platform access
Created on mobile data, separate email, different browser profile. Age accounts for 2+ weeks before use. No cross-linking to real identity.
Never visit targets directly
Use Google Cache, archive.ph, Startpage for Google proxying. Your IP appearing in target's logs exposes the investigation.
// WHAT EXPOSES YOU
  • LinkedIn profile view while logged in — target gets notification you viewed them
  • Direct target website visit — your IP in server, CDN, and WAF logs
  • Google search while signed in — all searches tied to identity
  • Browser fingerprinting — canvas, WebGL, fonts, resolution are unique
  • File metadata in shared files — machine username, timezone, software
  • Stylometry — your writing patterns identify you across platforms
  • Post timing patterns — when you post reveals timezone and schedule
📁 Documentation Standards
INTERMEDIATE
  • Hunchly — Chrome extension that auto-captures every visited page with timestamp. Creates court-ready reports automatically. Essential for all serious OSINT work.
  • CherryTree or Obsidian — Hierarchical encrypted notes for investigations. Organize by: target → category → finding → evidence. Free and fully offline.
  • Archive.ph + Wayback Machine — Archive target pages before they disappear. Social media posts get deleted within hours during incidents.
  • Screenshot everything with URL bar visible — shows source and timestamp. Never rely on memory or text notes alone for evidence.
  • Hash all evidence files immediately — SHA256 every screenshot, PCAP, and document as soon as collected. Proves integrity later.
  • Document negative results too — what you didn't find is also intelligence. Record where you looked and what was absent.
